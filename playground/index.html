Triage

Red-flag rules (instant, no GPU) → emergency. Non-red-flag cases get an LLM urgency opinion. Works with the LLM offline (rules only).

Content moderation

Toxicity (curated keywords) + distress detection in one call. Distress patterns escalate to a human; with the LLM off it fails safe (over-escalates).

Misinformation check (RAG)

Retrieves trusted-source evidence (WHO/CDC/…) and the LLM judges the claim against it, with citations. Needs a local LLM — without one it returns unverified + the evidence for human review.

SOAP note generation

From a consultation transcript → SOAP sections + extracted entities + ICD codes (mapped from config, not invented). Needs a local LLM; degrades to a marked draft otherwise.

Patient visit summary

Plain-language summary for the patient, with next steps and when to seek help. Needs a local LLM.

Support assist

Routes a ticket + drafts a reply for a human agent. Never gives medical advice; a distress pattern forces urgent + escalation even with the LLM off.

Drug-interaction check

Curated rule base is authoritative (brand names + dosage text tolerated); the optional LLM pass only adds candidates. It never says “safe” — absence of a flag means “not in our reference set”. Always clinician-confirmed.

Pre-consultation symptom intake

Red-flag triage net is authoritative for urgency (LLM can’t downgrade an emergency); the LLM only structures the complaint + asks clarifying questions. Advisory, never a diagnosis. Degrades to a rules skeleton with the LLM off.

Voice transcription (Whisper)

Upload a consultation clip → transcript. This runs the same code as the /transcribe task, in-process. First run loads the Whisper model (a moment). With stereo speaker-split on, a 2-channel recording is labelled by speaker.

No LLM needed — Whisper is self-hosted. base on CPU is slow on long audio; use a short clip to try it.

Semantic search

Embedding search over the content index (seeded with demo articles; add your own in “Add content”). Runs on CPU, no LLM.

Recommendations

Personalized content from a user profile, with a reason per item. CPU, no LLM.

Add your content

Push your own items into the index, then search/recommend over them. This is how the backend would feed real content (the AI service never reads your DB).

Paste a JSON array: [{"id":"x1","text":"title + body","type":"article","metadata":{"title":"…"}}]
